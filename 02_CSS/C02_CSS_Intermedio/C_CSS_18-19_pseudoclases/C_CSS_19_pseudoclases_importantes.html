IS

Is funciona para agrupar varios elementos a los que se le quiera modificar alguna propiedad, para usarlo se necesita la siguiente sintaxsis "etiqueta-padre :is(elementos-a-modificar) { }", por ejemplo, vamos a tener varios "li" y solo algunos van a tener un "class" para identificarlos y además, solo quiero modificar estos que se encuentres dentro de la etiqueta "section"

Es decir, a los elemento 2 y 4 que tienen el atributo "class" para identificarlo y además se encuentran dentro de la etiqueta section le vamos a cambiar el color a rojo.

Where

Where funciona igual que Is, sin embargo, tiene una especificidad (prioridad) menor.

Has

Has funciona para comprobar que tenga un elemento especificado como hijo (directo o indirecto) y en caso de que se cumpla esta condición se modificarán las propiedades. Por ejemplo:

  • Esta oración se encuentra dentro de un span, por lo tanto, debería cambiarme el color a rojo porque así lo estoy especificando.

  • En cambio este párrafo está fuera de ese span, por lo tanto, no se ve modificado el color de la letra, sin embargo, has tiene otra función y es la de modificar un elemento especificano un elemento posterior, por ejemplo, este párrafo está antes de un "div" vacio, y quiero que este cambie el fondo mediante esa condicion, para eso lo hago de la siguiente manera: li:has(+div).

Otro ejemplo del has

El has se puede usar de maneras interesantes como la que veremos, en el capítulo anterior vimos que se podía cambiar el color de un "input" para comprobar que lo que se estaba introduciendo fuera correcto, ahora lo que haremos será darle este cambio de color pero a su padre "label".

Para lograr eso debemos usar la siguiente sintaxsis:

label:has(input[type="email"]:valid){ }

Donde "label" es la etiqueta que se va a modificar, "has" comprueba que "label" tenga dentro un "input" y "valid" comprueba que el contenido sea válido
